11.1 基础像素单位（Pixel Unit）
选定：4px 基础像素单位
三个候选值的对比：
· 4px：在 1080×640 游戏窗口内 1px 网格点精确对齐，像素感清晰，星点（2×2 = 8px 渲染）刚好在视觉噪点范围之上可见；16×16 樱花 SVG viewBox 恰好对齐 4 的倍数网格；HiDPI（2x）下自然放大为 8px 物理像素，完整清晰。
· 6px：在大尺寸元素上效果好，但 1080px 无法被 6 整除，网格线位置会出现 0.5px 亚像素锯齿。
· 8px：像素块过大，星点在背景中太粗旷，与萌系细腻气质冲突。

结论：统一使用 4px 为像素原子单位。所有像素层元素的尺寸、间距均为 4 的倍数（4 / 8 / 12 / 16 / 24 / 32px）。

12.1 Synthwave 透视网格（Layer 1 核心）
Q6 决策：选用 Canvas 2D 方案
选型理由：Canvas 2D drawLine + requestAnimationFrame，而非 CSS perspective + rotateX。

CSS 方案的问题：
· perspective + rotateX 变换的是平面 DOM 元素（如设置了 background: repeating-linear-gradient 的 div）。网格线条之间的间距在透视变换后会均匀收缩，但 CSS 渐变的间距是固定 CSS 像素，无法实现真正的透视汇聚（近大远小、近疏远密），视觉上是梯形变形而不是透视纵深。
· 在 2K 高 DPR 屏上，CSS 渐变线在旋转后产生亚像素抗锯齿，网格线呈模糊灰边，违背像素清晰原则（PRD A16 验收项）。
· CSS 方案的 animation（translateZ 或 perspective-origin）驱动的"推进感"会触发整个层的 composite，比 Canvas 单帧绘制反而更耗 GPU 层内存。

Canvas 方案的优势：
· JS 精确计算每条线的起点/终点坐标，真正实现透视数学（vanishing point 汇聚），视觉效果正确。
· ctx.lineWidth 配合 devicePixelRatio 可在 HiDPI 下保持 1 物理像素锐利线条，满足 A16。
· 滚动推进用 requestAnimationFrame 驱动 offset 变量，帧预算完全可控，可在耗时超预算时跳帧降级。
· 无额外依赖，纯 Canvas 2D API，符合"无构建工具"约束。

性能兜底：Canvas 元素设置 will-change: transform，浏览器将其提升为独立合成层，与 UI DOM 树的重绘完全解耦。

网格视觉参数

/* Canvas 尺寸：与游戏窗口等大 */ width: 1080px (×devicePixelRatio) height: 640px (×devicePixelRatio) /* 消失点（vanishing point）*/ vp.x = 540 /* 水平居中 */ vp.y = 256 /* 垂直 40% 处（640×0.40）*/ /* 地平线在画面中上方，产生从脚下 延伸到远方的驾驶感 */ /* 纵向线（汇聚至消失点）*/ count: 12 条 间距: 从底部均匀分布，顶部汇聚 线宽: 1px（ctx 实际 devicePixelRatio 倍） 颜色: --pixel-grid-line #7B2FBE opacity: 18%（普通线） 地平线高光: --pixel-neon-purple #BF5FFF opacity: 35% /* 横向线（等间距，模拟地面进深）*/ count: 8 条（仅绘制地平线以下部分） 间距算法: 非线性（越接近画面底部间距越大） y[i] = vp.y + (height - vp.y) * (i/n)^1.8 颜色: 同纵向线（--pixel-grid-line）

12.4 像素噪点方案
选定：SVG feTurbulence filter 方案（不引入数据流字符雨）
数据流字符雨（Matrix 风格）：不引入。原因：绿色调与 B 站粉蓝主色冲突（PRD 方案 D 已排除），且大量字符 DOM 会超出 30 元素性能预算，与萌系气质不符。

像素噪点使用 SVG feTurbulence filter，通过 CSS 背景叠加实现：
/* 方法：SVG filter 定义为内联 SVG，通过 CSS filter 或 background 引用 */ /* 内联 SVG（嵌入 HTML，不引用外部文件）*/ <svg width="0" height="0" style="position:absolute"> <filter id="pixel-noise"> <feTurbulence type="fractalNoise" baseFrequency="0.80" /* 高频 = 细颗粒感，接近像素噪点 */ numOctaves="4" stitchTiles="stitch" /* 平铺无缝 */ result="noise"/> <feColorMatrix type="saturate" values="0" /* 转为灰度 */ in="noise" result="grayNoise"/> <feBlend in="SourceGraphic" in2="grayNoise" mode="multiply" result="blend"/> <feComponentTransfer> <feFuncA type="linear" slope="0.04"/> /* opacity ≤ 4% */ </feComponentTransfer> </filter> </svg> /* Layer 3 噪点层 div */ .bg-noise { position: fixed; inset: 0; z-index: 3; /* 与扫描线同层，合并为一个元素节省层 */ pointer-events: none; filter: url(#pixel-noise); opacity: 0.04; /* 整体不透明度 ≤ 4%，仅背景可见 */ background: rgba(255,255,255,0.02); /* 微白底让 multiply 有效 */ }
12.5 不采用的元素清单
  • 故障艺术（Glitch）常驻效果：CSS translateX 微偏移 + R/G/B 通道分离视觉上极易干扰文字阅读，游戏中大量数字 HUD（得分、目标分、次数）若叠加 glitch 效果会使玩家误读数值。MVP 阶段排除，仅作 P2 可选项（触发时长 ≤ 0.3s）。
  • 常驻全息光斑（Holographic Flare）：多个径向渐变光斑常驻会使背景过于"满"，与前景 chibi 争夺视觉权重，且在透视网格 + 扫描线 + 噪点已有三层背景的情况下，再加光斑会造成视觉混乱。保留为大牌触发的瞬时效果（≤ 0.5s），不常驻。
  • WebGL / Three.js 渲染：引入第三方依赖，破坏单文件纯静态约束；首屏加载时间增加 ≥ 200ms；Canvas 2D 完全可以实现所需效果。
  • 动态滚动扫描线：GPU 合成层开销较静态方案高 3–5 倍（每帧需重绘 repeating-gradient 动画），长时间观看产生视觉疲劳，PRD 明确排除。
  • 数据流字符雨：绿色调（Matrix 风格）与 B 站粉蓝色板强烈冲突；字符 DOM 数量（≥ 50 个）超出性能预算；风格与萌系二次元气质相悖。
  • 背景层 setInterval 驱动：PRD 明确要求所有背景动画使用 CSS animation 或 requestAnimationFrame，setInterval 无法与浏览器帧率同步，高刷新率屏幕（120Hz）下过度绘制，低端设备上掉帧明显。

14.5 V2.1 工程交班补充（容易翻车的地方）

给开发同事的 V2.1 新增备忘

  • ⑦ Canvas 必须处理 devicePixelRatio：不乘 dpr 在 Retina 屏（2x/3x）上网格线会模糊。正确做法：canvas.width = 1080 * dpr; canvas.height = 640 * dpr; ctx.scale(dpr, dpr); 之后所有 drawLine 坐标仍按 1080×640 的逻辑坐标写，浏览器自动放大。
  • ⑧ 像素星点 div 不要用 border-radius：2×2px div 加圆角会变成圆点而非方块，失去像素感。border-radius: 0 !important，配合 image-rendering: pixelated。
  • ⑨ 像素樱花 SVG 必须用 shape-rendering="crispEdges"：否则浏览器对 SVG rect 边缘做亚像素平滑，像素块变成模糊色块，像素感消失。
  • ⑩ Canvas 与 DOM 的层叠上下文：Canvas 用 position: fixed，它会创建新的 stacking context。注意 CSS will-change 也会创建 stacking context——背景层的 z-index 0/1/2/3 是相对于 html 根上下文的，#game 容器的 z-index 需明确设置（例如 position: relative; z-index: 10）确保高于背景层。
  • ⑪ 扫描线 + 噪点合并为一个 div：不要分两个 div。background 属性可以多层：background: repeating-linear-gradient(...), url(#noise-filter);，减少一个 DOM 层和一个合成层。
  • ⑫ 背景层禁止 setInterval：PRD 明确规定。星点闪烁用 CSS animation，樱花飘落用 CSS animation，网格推进用 rAF。不用 setInterval 驱动任何背景 DOM 操作。
